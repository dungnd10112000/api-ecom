## Commands

**Run the development server:**
```powershell
.venv\Scripts\python.exe -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

**Install dependencies:**
```powershell
.venv\Scripts\pip.exe install -r requirements.txt
```

**Environment setup:** Copy `.env.example` to `.env` (or create `.env`) with PostgreSQL credentials:
```
DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=yourpassword
DB_NAME=TCT_CRM
DB_SCHEMA=public
APP_PORT=8000
```

**API docs:** `http://localhost:8000/docs` (Swagger UI auto-generated by FastAPI)

## Architecture

This is a **read-only analytics/statistics API** built with FastAPI + SQLAlchemy (ORM, no raw SQL) connecting to an existing PostgreSQL CRM database (`TCT_CRM`). It does **not** own the database schema — it reflects the existing TCT_CRM tables.

### Data flow
```
PostgreSQL (TCT_CRM) → SQLAlchemy models (app/models.py) → routers → Pydantic schemas (app/schemas.py) → JSON response
```

### Key files
- `app/main.py` — FastAPI app setup, router registration, CORS, static file serving
- `app/config.py` — Settings via `pydantic-settings` loaded from `.env`
- `app/database.py` — SQLAlchemy engine, `SessionLocal`, `Base`, `get_db()` dependency
- `app/models.py` — All ORM models reflecting CRM tables
- `app/schemas.py` — Pydantic response models
- `app/routers/` — One file per domain, all prefixed `/api/<domain>/stats`

### Domain routers
| Router | Prefix | Domain |
|---|---|---|
| `leads.py` | `/api/leads/stats` | Lead pipeline |
| `opportunities.py` | `/api/opportunities/stats` | Sales opportunities |
| `orders.py` | `/api/orders/stats` | Order analytics |
| `customers.py` | `/api/customers/stats` | Customer analytics |
| `conversions.py` | `/api/conversions/stats` | Lead→Opportunity→Order funnel |
| `activities.py` | `/api/activities/stats` | CRM activities |
| `reports.py` | `/api/reports` | Aggregated reports |
| `sync.py` | `/api/sync` | Data sync utilities |
| `lists.py` | `/api/lists` | Lookup lists |

### Database model relationships
- `Lead` → `Opportunity` → `Quotation` → `Order` is the core sales funnel
- `Quotation` links to `Customer` via `PartnerId`
- `Quotation` ↔ `Product` is many-to-many via `LinkQuotationProduct`
- `Quotation` and `Order` are joined on `SoHopDong` (contract number string), not a FK
- `Taxonomy` is a generic lookup table: `TaxonomyType=1` = Area/Province, `TaxonomyType=3` = Lead Source
- Vietnamese field names: `NgayTao` = created date, `NgayCapNhat` = updated date, `TinhTrang` = status, `TrangThai` = state

### Common patterns
- All endpoints accept optional `date_from`/`date_to` query params for filtering
- `get_date_trunc_field()` in `opportunities.py` is a shared helper used across routers for `group_by=day|week|month` time bucketing
- `DB_SCHEMA` in config controls which PostgreSQL schema tables are mapped to (default: `public`)
